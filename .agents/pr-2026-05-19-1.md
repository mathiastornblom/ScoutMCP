### Bugs found and fixed during this QA run

**Bug 1 — Wrong ping path (health.ts)**  
`pingDirect()` used `${baseUrl}/rest/ping` which returns 404 on this server deployment.  
Actual working path is `/rest/api/v1/ping`. Fixed in `health.ts`.

**Bug 2 — Node.js v26 rejects `dispatcher` option on global `fetch`**  
Code passed `{ dispatcher: agent }` to Node's built-in `fetch()`. Node v26 does not support this option — throws `TypeError: fetch failed`. Fixed in `client.ts`, `health.ts`, and `tests/integration.ts` by switching to undici's own `fetch` import, which accepts `dispatcher` natively.

**Bug 3 — Root OU test assertion too narrow**  
`ou_get mode=root` test accepted `'not found'` in error text but the actual error was `"Request failed (HTTP 404)"`. Fixed assertion to also match `'404'`.
